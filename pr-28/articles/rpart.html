Published

May 24, 2026

パッケージの概要

rpart (Recursive Partitioning And Regression Trees)は、再帰的分割（データをグループに分けることを繰り返して木構造を作る手法）によって回帰木・分類木を構築するためのパッケージです。また、rpart.plot パッケージを用いることで、決定木をより見やすい形で可視化することができます。

使用例：Bostonデータ

MASSパッケージに含まれるBostonデータを用いて、住宅価格の中央値を推定する回帰木の構築を試してみます。

モデルの可視化

モデルを可視化します。plotで木の構造（分岐）を表示し、textで各ノードの分岐の基準や分類ラベルを表示します。

plot(model)
text(model)

rpart.plotを用いると、分類木をより分かりやすく表示させることができます。

rpart.plot(model)

モデル構築2（訓練データとテストデータに分割）

Bostonデータをモデル生成のための訓練データと、モデル評価のためのテストデータに分割します。データ割合は訓練データを7割、テストデータを3割とします。確認のため、データサイズを出力します。

# シードを設定
set.seed(123) 

# データの分割
sample_indices <- sample(1:nrow(df), 0.7 * nrow(df))  
df.train <- df[sample_indices, ]
df.test <- df[-sample_indices, ]

# データサイズの確認
c(nrow(df), nrow(df.train), nrow(df.test))
[1] 506 354 152

ハイパーパラメーターのチューニング

rpartの主なハイパーパラメーターは以下の通りです。

  • 木の複雑度に関するパラメータ(cp)
  • ノード分割の最小サンプル数(minsplit)
  • 木の最大の深さ(maxdepth)

これらのハイパーパラメータの最適な設定を探す作業がハイパーパラメーターのチューニングとなります。

各ハイパーパラメータは相互に影響を及ぼすため、ハイパーパラメータのチューニングは個別のパラメータごとに行うのではなく、対象のパラメータを同時に行う必要があります。ここではcpとminsplitのグリッドサーチにより、最適なパラメータを探す作業を行います。

data.frameの出力整形用の関数を定義します。

library(knitr)
library(kableExtra)

show_df <- function(df, cap=""){
  if(is_html_output()){
    kable(df, format = "html", caption = cap) %>%
      kable_styling(bootstrap_options = "striped", full_width = FALSE, position = "left")
  }else{
    kable(df, format = "pipe", caption = cap) 
  }

チューニング候補値です。expand.gridを作って全ての組み合わせを列挙します。

param_grid <- expand.grid(
  cp = c(0.01, 0.03, 0.05),
  minsplit = c(10, 20, 30, 40, 50)
)

各パラメータでモデル精度を検証してゆきます。

#結果保存用
results <- data.frame(
  cp = numeric(),
  minsplit = integer(),
  RMSE = numeric(),
  R2_Score = numeric()
)

set.seed(123)
for (i in 1:nrow(param_grid)) {
  params <- param_grid[i, ]
  
  model3 <- rpart(
    medv ~ .,
    data = df.train,
    control = rpart.control(cp = params$cp, minsplit = params$minsplit),
  )
  
  res_pred <- predict(model3, df.test, type = "vector")
  
  results <- rbind(results, data.frame(
    cp = params$cp,
    minsplit = params$minsplit,
    RMSE = round(RMSE(res_act, res_pred), 3),
    R2_Score = round(R2_Score(res_act, res_pred), 3)
  ))
}

show_df(results)

RMSEが小さく決定係数が大きな組み合わせであるcp = 0.01、minsplit = 10の設定が良さそうであることが分かりました。より汎化性能を高めるためには、クロスバリデーションを行う必要があります。

このモデルの分類木を可視化してみましょう。木の形も少し変わっています。

# シードを設定
set.seed(123)

model4 <- rpart(
  medv ~ ., 
  data = df.train,
  control = rpart.control(cp = 0.01, minsplit = 10)
)

rpart.plot(model4)
